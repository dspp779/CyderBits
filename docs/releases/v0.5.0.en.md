# Cyder 0.5.0

## TL;DR

Cyder 0.5.0 focuses on two major features:

- **Power Saving mode:** On Apple Silicon Macs, testing showed roughly a **90% reduction in Wine CPU energy usage**, making it especially useful for leaving online games running or idling for long periods.
- **Game Library:** Each game can now remember its own Wine and display settings, so you no longer need to reconfigure Cyder whenever you switch games.
- **New app logo:** Cyder now uses a new original visual identity to reduce potential trademark and copyright concerns.

## Highlights

### Power Saving mode

- Each game can use either Standard or Power Saving mode.
- On Apple Silicon Macs, Power Saving mode can deliver extremely low energy usage. In testing, Wine CPU energy consumption dropped by approximately 90%.
- It is particularly suitable for online-game AFK sessions, long idle periods, and games that do not require full CPU performance.
- Power Saving mode limits the CPU resources available to Wine. Standard mode is still recommended when high frame rates or low latency matter.

### Game Library and per-game settings

- The Game Library is now officially available. You can add Windows .exe files, double-click to launch them, open their launch options, or remove them from the library.
- Every game can save its own MSync/ESync, Retina Mode, DPI, font, font smoothing, energy mode, environment variables, and command-line arguments.
- Different games can use different system configurations. For example:

  - **Age of Empires II** is recommended to run at 96 DPI.
  - **Metal Slug** only works correctly at 96 DPI.
  - **BlueCG** can use Retina Mode with 192 DPI for a sharper launcher UI.

- Once applied, the game-specific settings are loaded automatically whether the EXE is opened from the Game Library, Finder, or directly. You no longer need to change global preferences every time you switch games.
- **Test** launches the game with the current draft while keeping the options window open, so you can compare, adjust, and test again. **Apply** saves the configuration.
- Launch options continue to use the shared prefix, so normal use does not require a separate bottle for every game.

### App icon

- Cyder now uses a new original app logo, reducing potential trademark and copyright concerns associated with the previous design.

## Implementation details

### Game Library and launch options

- The Game Library uses a fixed five-column, leading-aligned icon layout with the Add button in the title bar.
- Per-game settings are stored under a stable ID derived from the EXE's canonical path. They do not depend on the filename and do not automatically imply a separate bottle.
- Cyder extracts and caches game icons from PE resources, falling back to a generic EXE icon when no usable icon is available.
- Environment variables use one KEY=VALUE entry per line. Command-line arguments are entered on one line after the EXE command, with quoting supported for arguments containing spaces.
- Information icons explain supported options. The launch-options title bar uses the game name and its logo when available.
- ClearType BGR has been removed from font-smoothing options; Off, Grayscale, and ClearType RGB remain available.
- The Game Library and Preferences can remain open in the main Cyder process while games launch through a separate Cyder launch process.

### Shared prefix and settings application

- When the prefix is inactive, Retina, DPI, font, and font-smoothing values are applied quickly by editing user.reg directly, without starting Wine just to apply settings.
- When the prefix is already running, a normal EXE launch skips registry application and opens the game immediately. The saved settings are applied on the next launch after the wineserver has stopped.
- wine reg is reserved for **Preferences > Advanced > Apply All Settings** and is no longer called during a normal EXE launch.
- The default session guard has been removed. Cyder no longer blocks a second game merely because another game is using the shared prefix or has different synchronization or energy settings.
- Native, Finder, and shell launch paths all load the same per-game settings, environment variables, and command-line arguments.

### Launch and diagnostics fixes

- Fixed the shell launcher reloading global settings and overwriting per-game Retina, DPI, and related values.
- Fixed successful fast settings application being reported as a failure when there was no wineserver to stop during cleanup.
- Fixed a Test launch creating a second Cyder process and incorrectly treating the still-running Game Library session as a previous crash.
- Settings logs now record the effective Retina, DPI, font, and font-smoothing values. Game launch logs record the effective profile, synchronization, and energy settings.

## Upgrading

1. Quit Cyder and all games launched through Cyder.
2. Replace the previous Cyder.app with Cyder 0.5.0.
3. Launch Cyder once and wait for its environment check to complete.
4. Add EXE files to the Game Library. Existing profile records are merged into the library automatically.

Existing engines, bottles/shared, profile bottles, global settings, and game files are preserved. Game Library metadata and per-game settings are stored in:

~~~text
~/Library/Application Support/Cyder/game-library.json
~/Library/Application Support/Cyder/settings.json
~~~

## Known limitations

- Icon extraction may fail for some games. Known examples include **Pikachu Volleyball**, **Age of Empires**, and **Richman 4**. The games can still be added and launched normally, but Cyder will show a generic EXE icon.
- Retina Mode, DPI, font, and font smoothing are prefix/wineserver-level registry settings. They cannot be switched while the shared prefix is running; newly launched games inherit the registry state already loaded by the current session.
- Allowing multiple games to launch does not isolate different registry configurations inside one wineserver. Fully isolated configurations still require separate prefixes.
- MSync/ESync compatibility within the same wineserver is ultimately determined by Wine. Cyder no longer blocks the second game in advance.
- Power Saving mode reduces available CPU performance. M1 Pro and M1 Max systems have fewer efficiency cores, so some games may become noticeably less responsive.
- If an EXE is moved, it must currently be added to the Game Library again. Security-scoped bookmarks are not yet stored.
- Cyder is still ad-hoc signed and is not yet signed with an Apple Developer ID or notarized.

## Validation

- Game Library, settings UI, per-game settings bridge, fast user.reg editing, launcher, session, and Finder lifecycle regression tests passed.
- Universal arm64/x86_64 app compilation and deep code-signature verification passed.
