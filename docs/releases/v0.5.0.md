# Cyder 0.5.0

Cyder 0.5.0 的兩項重點是省電模式與遊戲庫。省電模式讓 Apple Silicon Mac 在長時間執行老遊戲時大幅降低能源消耗；遊戲庫則會記住每款遊戲需要的系統配置，不必在切換遊戲時反覆修改全域設定。

## 更新重點

### 省電模式

- 每款遊戲都能選擇「標準」或「省電」能源模式。
- 省電模式在 Apple Silicon 晶片的電腦上可達到極致省電效果；實測 Wine CPU 能源消耗約減少九成。
- 適合線上遊戲掛機、長時間待機，或不需要完整 CPU 效能的遊戲情境。
- 省電模式會限制 Wine 使用的 CPU 資源，因此需要高幀率或低延遲時仍建議使用標準模式。

### 遊戲庫與每遊戲設定

- 正式開放遊戲庫，可加入 Windows `.exe`、雙擊啟動、開啟啟動選項或移除遊戲。
- 每款遊戲都能保存自己的 MSync／ESync、Retina Mode、DPI、字體、字體平滑、能源模式、環境變數與命令列參數。
- 不同遊戲可使用不同系統配置，例如：
  - 《世紀帝國 2》建議使用 96 DPI。
  - 《越南大戰（合金彈頭）》只能使用 96 DPI。
  - 《水藍魔力》可使用 Retina Mode 與 192 DPI，改善啟動器的顯示畫質。
- 設定套用後，從遊戲庫、Finder 或直接開啟該 EXE 都會自動載入；使用者不必每次更換遊戲時重新調整偏好設定。
- 「測試」會以目前草稿啟動遊戲並保留選項視窗，方便比較、調整與再次測試；「套用」才會保存設定。
- 啟動選項直接使用共用 prefix，一般使用不需要為每款遊戲建立獨立 bottle。

### App 圖示

- 更新 Cyder app logo，使用新的原創識別設計，降低可能的商標與著作權疑慮。

## 背後實作細節

### 遊戲庫與啟動選項

- 遊戲庫使用固定五欄、靠左排列的圖示介面；加入按鈕位於標題列。
- 每遊戲設定以 EXE canonical path 衍生的穩定 ID 保存，不依賴檔名，也不代表一定建立獨立 bottle。
- 從 PE 資源擷取並快取遊戲圖示；沒有可用圖示時使用通用 EXE 圖示。
- 環境變數使用每行 `KEY=VALUE` 輸入；命令列參數以單行直接接在 EXE 後，支援引號保留含空白參數。
- 各選項提供資訊圖示與說明；視窗標題使用遊戲名稱與可用的遊戲 logo。
- 移除字體平滑的 ClearType BGR 選項，保留關閉、灰階與 ClearType RGB。
- 遊戲庫與偏好設定可在同一 Cyder 行程中切換；啟動遊戲則交由獨立的 Cyder launch process，遊戲庫保持可用。

### 共享 prefix 與設定套用

- prefix 未執行時，Retina、DPI、字體與平滑設定會直接快速修改 `user.reg`，避免為套用設定啟動 Wine。
- prefix 已在執行時，一般 EXE 啟動會略過 registry 套用並直接開啟；設定保留至 wineserver 停止後的下一次啟動再套用。
- `wine reg` 僅用於「偏好設定 → 進階 → 套用所有設定」，不再由一般 EXE 啟動流程呼叫。
- 移除預設 session guard，不再因另一個遊戲使用共用 prefix 或採用不同同步／能源設定而拒絕啟動。
- Native、Finder／shell 啟動路徑都會載入相同的每遊戲設定、環境變數與命令列參數。

### 啟動與診斷修正

- 修正 shell launcher 重讀全域設定、覆蓋每遊戲 Retina／DPI 等選項的問題。
- 修正快速設定成功後，因 wineserver cleanup 沒有可停止的 server 而被誤判失敗的問題。
- 修正「測試」建立第二個 Cyder 行程時，把仍在執行的遊戲庫 session 誤判為前次異常中斷的問題。
- 設定套用 log 會記錄實際使用的 Retina、DPI、字體與平滑值；遊戲啟動 log 會記錄有效的 profile、同步與能源設定。

## 升級方式

1. 關閉 Cyder 與所有由 Cyder 啟動的遊戲。
2. 以 0.5.0 的 `Cyder.app` 取代舊版 App。
3. 單獨啟動一次 Cyder，等待環境檢查完成。
4. 在遊戲庫加入 EXE；既有 Profile 記錄會自動合併至遊戲庫。

既有 engine、`bottles/shared`、Profile bottle、全域設定與遊戲檔案都會保留。遊戲庫資料與每遊戲設定分別儲存在：

```text
~/Library/Application Support/Cyder/game-library.json
~/Library/Application Support/Cyder/settings.json
```

## 已知限制

- 部分遊戲可能無法從 EXE 擷取 logo；目前已知包括《皮卡丘排球》、《世紀帝國》與《大富翁 4》。擷取失敗時遊戲仍可正常加入與啟動，但會顯示通用 EXE 圖示。
- Retina Mode、DPI、字體與字體平滑是 prefix／wineserver 層級設定。共享 prefix 已在執行時無法切換；新啟動的遊戲會沿用目前 session 的 registry 狀態。
- 允許同時啟動多個遊戲不代表不同 registry 設定能在同一 wineserver 中隔離。需要完全隔離時仍須使用不同 prefix。
- MSync／ESync 的組合與同一 wineserver 的實際相容性仍由 Wine 決定；Cyder 不再預先阻擋第二個遊戲。
- 省電模式會降低可用 CPU 效能；M1 Pro／Max 的節能核心數量較少，部分遊戲可能明顯卡頓。
- EXE 移動後需要重新加入遊戲庫；目前未保存 security-scoped bookmark。
- 本版本仍採 ad-hoc 簽署，尚未使用 Apple Developer ID 與 notarization。

## 驗證結果

- 遊戲庫、設定 UI、每遊戲設定橋接、快速 `user.reg`、launcher、session 與 Finder lifecycle 回歸測試通過。
- Universal `arm64`／`x86_64` app 編譯與深度簽章驗證通過。
