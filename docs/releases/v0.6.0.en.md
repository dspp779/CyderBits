# Cyder 0.6.0

## TL;DR

Cyder 0.6.0 focuses on general-purpose install and launch quality:

- **Native Winetricks** component installation from Cyder settings
- **Leaner launch diagnostics** and health probes
- **Per-launch dynamic command-line arguments** without public Cyder CLI options
- **Smaller, more reproducible engine packaging** with bundled universal zstd, strip, and safer MoltenVK selection
- **Engine version label/slug equality**, so extraction no longer leaves document launches stuck as “not ready”

This release intentionally **does not** include MapleStory OEM / CX26 special compatibility work.

## Highlights

### Native Winetricks

- Install common Winetricks components from the settings UI without opening a separate terminal.
- The app bundles the Winetricks script and license files alongside Cyder’s bottle support path.

### Launch performance and health checks

- Reduces noisy launch logs and health probes for everyday game starts.
- Successful bottle rebuilds discard rollback copies so temporary space is not left behind.

### Per-launch dynamic arguments

- Cyder.app does not keep public command-line options. Arguments received via Finder / `open --args` or direct argv apply to that launch only.
- When no dynamic arguments are provided, Cyder still uses the arguments saved in the Game Library / settings.
- Optional `CYDER_REDACT_DYNAMIC_ARGS=1` redacts dynamic argument values in diagnostic logs.

### Engine packaging and size

- Bundles a universal zstd CLI so packing and extraction no longer depend on a separately installed system zstd.
- `strip-wine-install.sh`, `pack-engine-artifact.sh`, and dylib bundling support leaner, reproducible runtime artifacts.
- When repacking an existing engine, `VULKAN_SOURCE=existing` preserves the already-tested MoltenVK instead of silently replacing it with Homebrew’s build.

### Engine version comparison fix

- Display labels such as `wine crossover 26.2.0 (wine 11.0)` and filesystem slugs such as `crossover-26.2.0-wine-11.0` are treated as the same engine version.
- After a successful extraction, document launches no longer permanently report that the environment is not ready.

## Implementation details

### Winetricks and settings UI

- Adds `scripts/cyder-winetricks.sh` and ships `tools/winetricks/` inside App Resources.
- The settings UI invokes installation through the existing launcher / support paths.

### Dynamic argv contract

- The native AppKit path and shell wrapper (`create-cyder-app.sh` → `cyder_launcher.sh --launch-exe … --`) share the same “per-launch arguments override saved arguments” semantics.
- Wine locale exports now also set `LC_CTYPE` alongside `LANG` / `LC_ALL`.

### Packaging toolchain

- Adds `build-universal-zstd.sh`, `build-media-stack.sh` (minimal GStreamer media stack when needed), and related docs.
- Runtime builds default to `--disable-tests`. CrossOver-FOSS MoltenVK installs verify the version manifest and ignore unverified caches.

## Upgrading

1. Quit Cyder and all games launched through Cyder.
2. Replace the previous Cyder.app with Cyder 0.6.0.
3. Launch Cyder once and wait for its environment check to complete.
4. Launch games from the Game Library or Finder as usual. For per-launch arguments, use `open -n -a Cyder.app game.exe --args …`.

Existing engines, bottles/shared, profile bottles, Game Library data, and per-game settings are preserved.

## Known limitations

- Cyder is still ad-hoc signed and is not yet signed with an Apple Developer ID or notarized.
- Dynamic arguments affect only the current launch and are not written back to saved Game Library arguments.
- MapleStory OEM / CX26 special compatibility and related patches are out of scope for this release.

## Validation

- Narrow tests passed for Winetricks, launcher, prefix rebuild, template bootstrap, dynamic argv, force settings UI, open-files lifecycle, engine version, sign-wine, universal zstd, strip, bundle dylib source, and app payload.
