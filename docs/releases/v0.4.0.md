# Cyder 0.4.0

Cyder 0.4.0 是一次以初始化可靠性、Shared bottle 穩定性與設定相容性為主的更新。所有遊戲仍預設共用 `bottles/shared`；需要獨立環境時，使用者仍可建立個別 Profile bottle。

本版本不包含遊戲庫功能，也不包含省電模式。兩者目前都暫時隱藏於使用者介面，待後續版本完成整合與驗證後再重新開放。

## 更新重點

### 初始化與環境可靠性

- 每次開啟 Cyder 都會檢查 Wine engine、prefix registry、`kernel32.dll` 與最小 Wine probe。
- `wineboot`、health check、設定套用與遊戲啟動各自保存獨立 operation log。
- 初始化錯誤會分類記錄 exit、signal、timeout 與必要產物缺失，並提供重試、重建環境與開啟記錄等處理方式。
- 重建 Windows 遊戲環境採 staging、backup、atomic switch 與 rollback，不會先刪除原有 Shared bottle。
- Wine runtime 安裝後會進行簽署，降低 Gatekeeper 阻擋 extracted runtime 的機率。

### Shared、Template 與 Profile

- 新增 versioned Pristine / Golden template lifecycle。
- Golden template 會準備固定版本的 Wine Mono 10.4.1、Wine Gecko 2.47.4 與 Cyder baseline settings。
- Shared bottle 與獨立 Profile bottle 都由 Golden template 建立。
- APFS 優先使用 clonefile；不支援時會退回一般 copy。
- Profile 以 EXE canonical path 建立穩定識別，不以檔名作為唯一 ID。
- 舊版 Shared bottle 與既有設定會保留，不會因升級自動刪除。

### 本版本暫不提供的功能

- 遊戲庫按鈕與遊戲庫介面暫時隱藏，不列入 0.4.0 的使用流程與功能承諾。
- 「遊戲設定」偏好設定分頁暫時隱藏，因此省電模式也不列入本版本。
- 遊戲庫與省電模式相關程式會在後續版本完成整合、驗證後再重新開放。

### 設定與相容性

- 全域顯示、Retina、DPI、字體、字體平滑與 MSync / ESync 設定仍可使用。
- 設定變更會立即保存；未執行中的 Shared bottle 會以快速路徑更新受 Cyder 管理的 registry 值。
- 新增 recipe schema 與 offline validate / plan / apply framework。
- 已建立 AoE II、越南大戰、大富翁 4、皮卡丘排球、BlueCG 與 LF2 的初版 recipe metadata。
- cnc-ddraw 與 LF2 外部元件尚未提供一鍵安裝，仍需後續確認來源、授權、checksum 與可重入安裝流程。

## 升級方式

1. 關閉所有由 Cyder 啟動的遊戲。
2. 以新版 `Cyder.app` 取代舊版 App。
3. 單獨啟動一次新版 Cyder，等待環境檢查完成。

既有 `bottles/shared`、engine、設定與遊戲檔案會繼續使用。

## 已知限制

- 本版本仍以 Shared bottle 為主要使用模式。
- 遊戲庫與省電模式尚未開放；相關功能預計於後續版本提供。
- Intel Mac 實機 smoke test、Finder 雙擊與長時間多遊戲回歸仍在持續驗收。

## Gatekeeper 說明

本版本仍採 ad-hoc 簽署，尚未使用 Apple Developer ID 與 notarization。第一次開啟時，macOS 可能需要使用者在 Finder 右鍵選擇「打開」，或在「系統設定 → 隱私權與安全性」允許執行。

## 驗證結果

- 本輪相關 Cyder shell、Profile、template、recipe、session、settings、Finder lifecycle、prefix rebuild 與 wineboot logging 測試通過。
- 缺少本機 `downloads/wine-mono-10.4.1-x86.msi` 的 bootstrap 測試會略過，不代表外部元件安裝已完成。
