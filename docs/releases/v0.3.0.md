# Cyder 0.3.0

Cyder 0.3.0 主要改善錯誤診斷、啟動可靠性，以及 Intel Mac 上發生閃退時的可追查性。過去可能無聲退出的 Wine、bootstrap 與 launcher 錯誤，現在會保留記錄並顯示可操作的錯誤對話框。

## 更新重點

### 錯誤診斷與記錄

- 每次啟動建立獨立 session log，記錄 App 版本、macOS 版本、CPU 架構與執行階段。
- 新增結構化錯誤資訊，包括錯誤代碼、發生階段、process exit code、signal 與相關 log。
- Wine 提前結束、遭 signal 終止或無法啟動時，不再無聲退出。
- App 上次若未正常結束，下次啟動會提示中斷階段並提供前次記錄。
- 錯誤對話框可直接「複製診斷資訊」或「開啟記錄」。
- 診斷內容會遮罩使用者家目錄路徑，並自動輪替舊 session log。

記錄預設存放於：

```text
~/Library/Application Support/Cyder/Logs/
├── sessions/
├── session-state.json
├── last-error.json
└── last-launch.log
```

### 啟動與 bootstrap 可靠性

- launcher 現在會分別記錄 Rosetta 檢查、engine 解壓、wineboot、字型、Wine Mono、libarchive、顯示設定與 EXE 啟動階段。
- bootstrap 任一子階段失敗都會正確中止，不再誤寫成功 marker。
- 每次啟動遊戲會建立獨立 log；`last-launch.log` 永遠指向最近一次啟動記錄。
- Finder 啟動時會在需要顯示警告或錯誤前切換至前景，避免對話框在背景不可見。
- 改善 macOS 14 以上的 Wine 視窗啟用流程，同時保留較舊 macOS 的相容路徑。
- DirectDraw override 改為套用至 `bluecg.exe`。

### App 與相容性

- Cyder launcher 同時包含 `arm64` 與 `x86_64`，最低支援 macOS 12.0。
- Apple Silicon Mac 透過 Rosetta 2 執行 x86_64 Wine engine。
- Intel Mac 直接執行 x86_64 Wine engine，不需要 Rosetta 2。
- 更新 Cyder 與 CyderBits 圖示。
- Wine engine 維持 `CX26.2.0-W11-Cyder003`，本次不更換 bottle 或 engine 格式。

## 升級方式

1. 關閉所有由 Cyder 啟動的遊戲。
2. 以新版 `Cyder.app` 取代舊版 App。
3. 直接啟動新版 Cyder；既有 shared bottle、設定與遊戲資料會繼續使用。

## Gatekeeper 說明

本版本目前採 ad-hoc 簽署，尚未使用 Apple Developer ID 與 notarization。第一次開啟時，macOS 可能阻擋 App 或提示無法驗證開發者。

可使用下列任一方式允許開啟：

- 在 Finder 對 `Cyder.app` 按右鍵，選擇「打開」，再確認一次。
- 開啟「系統設定 → 隱私權與安全性」，在安全性區域選擇「仍要打開」。

請只對從可信來源取得、且 checksum 相符的檔案執行上述操作。

## 驗證結果

- Apple Silicon 與 Intel Mach-O 架構檢查通過。
- 最低部署版本確認為 macOS 12.0。
- App 深度簽章驗證通過。

## 回報問題

若 Cyder 顯示錯誤，請附上：

- 錯誤對話框中的錯誤代碼與發生階段。
- 「複製診斷資訊」取得的內容。
- 對應的 session log 或 `last-launch.log`。
- Mac 型號、Intel／Apple Silicon，以及 macOS 完整版本。
