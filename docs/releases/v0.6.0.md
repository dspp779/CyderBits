# Cyder 0.6.0

Cyder 0.6.0 聚焦一般用途的安裝與啟動體驗：內建 Winetricks、更精簡的啟動診斷、當次動態命令列參數、CX26.3 engine 與可重現的 packaging，以及對齊 **macOS 10.15** 的 runtime 地板。本版本**不含** MapleStory OEM／CX26 special 相容性路徑。

**引擎標籤：** `CX26.3.0-W11-Cyder004`

## 更新重點

### 原生 Winetricks

- 設定介面可直接安裝常見 Winetricks 元件，不必另開終端機。
- App 內附 Winetricks 腳本與授權檔，安裝路徑與 Cyder bottle 流程對齊。

### 啟動效能與健康檢查

- 減少啟動過程中的冗餘 log 與 health probe，縮短日常開啟遊戲的等待與噪音。
- Bottle rebuild 成功後會丟棄 rollback 複本，避免殘留占用空間。
- Bootstrap／provision 會寫入進度文字；Swift UI 與 macOS &lt; 12 的 osascript 路徑都會顯示。
- Golden baseline 改為單次 `regedit /s` 匯入，縮短建立環境時間。

### 當次動態命令列參數

- Cyder.app 不保留公開命令列選項；透過 Finder／`open --args` 或直接 argv 傳入的參數，只影響該次啟動。
- 未傳入動態參數時（含僅開啟 EXE、argv 為空），仍使用遊戲庫／設定中保存的命令列參數，不會清空。
- 可選 `CYDER_REDACT_DYNAMIC_ARGS=1` 在診斷 log 中遮罩動態參數內容。

### 啟動選項 UI

- 遊戲「啟動選項」視窗改為較寬、較矮（約 520×450）；環境變數與命令列參數欄位加寬變矮，並加上 placeholder。
- 環境變數支援 `KEY=value`，以空白或換行分隔（換行視同空白）；值含空白可用引號。
- 命令列參數改為多行輸入，換行視同空白；含空白參數仍可用引號保留。

### Engine（CX26.3）與 packaging

- 預設引擎升級為 CrossOver 26.3.0 系 artifact：`CX26.3.0-W11-Cyder004`。
- 內建 universal zstd；`strip-wine-install.sh`、`pack-engine-artifact.sh` 與 dylib bundling 支援更精簡、可重現的 runtime。
- 重新打包時預設 `VULKAN_MODE=with` + `VULKAN_SOURCE=existing`，保留已驗證的 MoltenVK。
- **Bundled Homebrew／media dylib**（freetype、gnutls 鏈、glib／gstreamer 等）以 `MACOSX_DEPLOYMENT_TARGET=10.15` **從原始碼**建置；打包後若 Mach-O `minos` &gt; 10.15 會失敗。不再依賴僅支援 macOS 14+ 的 brew bottles。

### macOS 系統版本

| 層級 | 最低版本 | 說明 |
|------|----------|------|
| Cyder.app（`LSMinimumSystemVersion`） | **10.15** | 與 Wine engine／bundled dylib 對齊 |
| Swift 設定／遊戲庫 UI | **12.0** | `CyderSwift` |
| 10.15–11.x | bash + osascript | Retina 強制關閉；顯示 bootstrap 進度 |
| Apple Silicon | **11.0** | 需要 Rosetta 2 |

### Engine 版本比對修正

- 修正顯示用 label 與檔案系統 slug 被當成不同版本的問題。
- 成功解壓後，文件啟動不會再誤報環境尚未就緒。

### 啟動診斷與字體預設

- 遊戲庫「測試」啟動會寫入 `Logs/last-launch.log`（CrossOver 風格 preamble：指令、MSync／ESync、環境與 Wine 輸出）。
- 若系統已有細明體（MingLiU）家族，預設字體改為 MingLiU；否則維持 Songti TC。

### 簽署（可選）

- `create-cyder-app.sh`／`sign-wine.sh` 支援 Developer ID + secure timestamp；本機可用 `SIGN_IDENTITY=-` 改回 ad-hoc。
- 說明見 [release-signing.zh-TW.md](../release-signing.zh-TW.md)。正式 notarization 與否依發行流程決定。

## 背後實作細節

### Winetricks 與設定 UI

- 新增 `scripts/cyder-winetricks.sh`，並將 `tools/winetricks/` 一併打進 App Resources。
- 設定 UI 透過既有 launcher／support 路徑呼叫安裝流程。

### 動態 argv 契約

- Native AppKit 路徑與 shell wrapper（`create-cyder-app.sh` → `cyder_launcher.sh --launch-exe … --`）共用同一套「當次參數覆寫已存參數」語意。
- Wine locale 匯出補上 `LC_CTYPE`，與 `LANG`／`LC_ALL` 一致。

### Packaging 工具鏈

- 新增 `build-universal-zstd.sh`、`build-media-stack.sh`（最小 GStreamer 媒體堆疊）與相關文件。
- `brew_x86_install_runtime`（`env-x86_64.sh`）對 runtime 套件做 source 建置與 10.15 compiler wrapper；gnutls 使用 vendored tap `homebrew/ogom-local`（避開 GitLab older-clang patch 403）。
- MoltenVK 可自 CrossOver.app 安裝（`install-crossover-app-moltenvk.sh`）或自 CX FOSS 建置；SPIRV-Cross 優先使用 tarball 內已 vendor 的複本。
- runtime build 預設 `--disable-tests`；crossover-foss MoltenVK 安裝會檢查 version manifest。

### Engine 升級與 bottle 重建

- Engine 版本變更時會重置 `bottles/shared`，並對**目前** engine 重新 `wineboot` 後安裝 mono／gecko／libarchive 與 baseline 設定。
- 不再沿用舊的 `templates/pristine`／`templates/golden` clone；升級時會刪除 Application Support 下的舊 `templates/`。
- 可版本化的 template bottles 機制延後至 1.0.0；在此之前 shared 與獨立 profile 都採直接 provision。

### Legacy UI

- `MacOS/Cyder` 永遠是薄 wrapper：≥12 轉 `CyderSwift`；更早則 `exec` osascript 進度 UI（`cyder-legacy-ui.applescript`）。

## 升級方式

1. 關閉 Cyder 與所有由 Cyder 啟動的遊戲。
2. 以 0.6.0 的 `Cyder.app` 取代舊版 App。
3. 若本機曾安裝同標籤但較舊的 engine 檔（dylib minos 仍為 14／15），請刪除後重開以重裝：
   `rm -rf ~/.cyder/runtime/Engines/wine-x86_64`
4. 單獨啟動一次 Cyder，等待環境檢查完成（engine 升級時會重建 shared）。
5. 照常從遊戲庫或 Finder 啟動遊戲；若需當次參數，使用 `open -n -a Cyder.app game.exe --args …`。

既有遊戲庫與每遊戲設定會保留。`bottles/shared` 在 engine 版本變更時會重建；獨立 profile bottle 不會被 engine 升級自動刪除。

## 已知限制

- 動態參數只影響當次啟動，不會寫回遊戲庫保存的命令列參數。
- MapleStory OEM／CX26 special 相容性與相關 patches 不在本版本範圍。
- Template bottles（pristine／golden）尚未恢復；獨立環境改為當場 wineboot + 元件安裝。
- macOS 10.15–11.x 無原生遊戲庫／完整設定 UI，僅 legacy 選檔與進度列。
- 正式 notarization 狀態依發行流程；見簽署文件。

## 驗證結果

- Winetricks、launcher、prefix rebuild、direct bootstrap、動態 argv、force settings UI、open-files lifecycle、engine version、sign-wine、universal zstd、strip、bundle dylib source／minos gate、legacy UI、env-x86_64 runtime install 與 app payload 窄測通過。
