# Cyder 0.6.0

Cyder 0.6.0 聚焦一般用途的安裝與啟動體驗：內建 Winetricks 元件安裝、更精簡的啟動診斷、當次動態命令列參數，以及更小、更可重現的 engine 打包流程。本版本**不含** MapleStory OEM／CX26 special 相容性路徑。

## 更新重點

### 原生 Winetricks

- 設定介面可直接安裝常見 Winetricks 元件，不必另開終端機。
- App 內附 Winetricks 腳本與授權檔，安裝路徑與 Cyder bottle 流程對齊。

### 啟動效能與健康檢查

- 減少啟動過程中的冗餘 log 與 health probe，縮短日常開啟遊戲的等待與噪音。
- Bottle rebuild 成功後會丟棄 rollback 複本，避免残留占用空間。

### 當次動態命令列參數

- Cyder.app 不保留公開命令列選項；透過 Finder／`open --args` 或直接 argv 傳入的參數，只影響該次啟動。
- 未傳入動態參數時，仍使用遊戲庫／設定中保存的命令列參數。
- 可選 `CYDER_REDACT_DYNAMIC_ARGS=1` 在診斷 log 中遮罩動態參數內容。

### Engine packaging 與體積

- 內建 universal zstd，打包與解壓不再依賴系統另裝的 zstd。
- `strip-wine-install.sh`、`pack-engine-artifact.sh` 與 dylib bundling 支援更精簡、可重現的 runtime artifact。
- 重新打包既有 engine 時，預設以 `VULKAN_SOURCE=existing` 保留已驗證的 MoltenVK，避免被 Homebrew 版本靜默取代。

### Engine 版本比對修正

- 修正顯示用 label（例如 `wine crossover 26.2.0 (wine 11.0)`）與檔案系統 slug（例如 `crossover-26.2.0-wine-11.0`）被當成不同版本的問題。
- 成功解壓後，文件啟動不會再誤報環境尚未就緒。

## 背後實作細節

### Winetricks 與設定 UI

- 新增 `scripts/cyder-winetricks.sh`，並將 `tools/winetricks/` 一併打進 App Resources。
- 設定 UI 透過既有 launcher／support 路徑呼叫安裝流程。

### 動態 argv 契約

- Native AppKit 路徑與 shell wrapper（`create-cyder-app.sh` → `cyder_launcher.sh --launch-exe … --`）共用同一套「當次參數覆寫已存參數」語意。
- Wine locale 匯出補上 `LC_CTYPE`，與 `LANG`／`LC_ALL` 一致。

### Packaging 工具鏈

- 新增 `build-universal-zstd.sh`、`build-media-stack.sh`（最小 GStreamer 媒體堆疊，供需要時建置）與相關文件。
- runtime build 預設 `--disable-tests`；MoltenVK crossover-foss 安裝會檢查 version manifest，略過未驗證快取。

### Engine 升級與 bottle 重建

- Engine 版本變更時會重置 `bottles/shared`，並對**目前** engine 重新 `wineboot` 後安裝 mono／gecko／libarchive 與 baseline 設定。
- 不再沿用舊的 `templates/pristine`／`templates/golden` clone；升級時會刪除 Application Support 下的舊 `templates/`，避免誤用過期 bottle。
- 可版本化的 template bottles 機制延後至 1.0.0；在此之前 shared 與獨立 profile 都採直接 provision。

## 升級方式

1. 關閉 Cyder 與所有由 Cyder 啟動的遊戲。
2. 以 0.6.0 的 `Cyder.app` 取代舊版 App。
3. 單獨啟動一次 Cyder，等待環境檢查完成（engine 升級時會重建 shared）。
4. 照常從遊戲庫或 Finder 啟動遊戲；若需當次參數，使用 `open -n -a Cyder.app game.exe --args …`。

既有遊戲庫與每遊戲設定會保留。`bottles/shared` 在 engine 版本變更時會重建；獨立 profile bottle 不會被 engine 升級自動刪除。

## 已知限制

- 動態參數只影響當次啟動，不會寫回遊戲庫保存的命令列參數。
- MapleStory OEM／CX26 special 相容性與相關 patches 不在本版本範圍。
- Template bottles（pristine／golden）尚未恢復；獨立環境改為當場 wineboot + 元件安裝。

## 驗證結果

- Winetricks、launcher、prefix rebuild、direct bootstrap、動態 argv、force settings UI、open-files lifecycle、engine version、sign-wine、universal zstd、strip、bundle dylib source 與 app payload 窄測通過。
